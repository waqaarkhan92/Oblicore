## 5. Accessibility Standards

### 5.1 WCAG 2.1 AA Compliance Checklist

| Criterion | Standard | Implementation |
|-----------|----------|----------------|
| 1.1.1 Non-text Content | Alt text for images | All images have descriptive alt attributes |
| 1.3.1 Info and Relationships | Semantic HTML | Proper heading hierarchy (h1-h6) |
| 1.4.1 Use of Color | Not color alone | Status uses icons + color + text |
| 1.4.3 Contrast | 4.5:1 minimum | All text meets contrast ratio |
| 1.4.4 Resize Text | 200% zoom | UI remains functional at 200% |
| 2.1.1 Keyboard | Full keyboard access | All interactive elements focusable |
| 2.1.2 No Keyboard Trap | Escape traps | Modal can be closed with Escape |
| 2.4.1 Bypass Blocks | Skip links | Skip to main content link |
| 2.4.3 Focus Order | Logical order | Tab order follows visual order |
| 2.4.4 Link Purpose | Clear links | Links describe destination |
| 2.4.6 Headings and Labels | Descriptive | All form fields have labels |
| 3.1.1 Language of Page | Lang attribute | `<html lang="en">` |
| 3.2.1 On Focus | No surprises | Focus doesn't trigger context change |
| 3.3.1 Error Identification | Error messages | Errors clearly identified |
| 3.3.2 Labels or Instructions | Form guidance | Required fields marked |
| 4.1.1 Parsing | Valid HTML | No duplicate IDs |
| 4.1.2 Name, Role, Value | ARIA labels | Custom components have ARIA |

### 5.2 Keyboard Navigation

| Key | Action |
|-----|--------|
| Tab | Move to next focusable element |
| Shift+Tab | Move to previous focusable element |
| Enter | Activate button/link |
| Space | Toggle checkbox, activate button |
| Arrow keys | Navigate within components (dropdowns, lists) |
| Escape | Close modal, cancel action |
| Home/End | First/last item in list |

### 5.3 Focus Indicators

```css
/* Focus ring specification */
:focus-visible {
  outline: 2px solid #3B82F6;  /* Blue-500 */
  outline-offset: 2px;
  border-radius: 4px;
}

/* High contrast mode support */
@media (prefers-contrast: high) {
  :focus-visible {
    outline: 3px solid currentColor;
  }
}
```

## 6. Error States & Handling

### 6.1 Error State Types

| Type | Visual | Recovery |
|------|--------|----------|
| Field Error | Red border + message below field | User corrects input |
| Form Error | Error summary at top of form | User reviews and corrects |
| API Error | Toast notification | Retry or contact support |
| Network Error | Full-page error state | Retry button |
| Permission Error | Modal with explanation | Request access or logout |
| Not Found | 404 page | Navigate home |
| Server Error | 500 page | Retry or contact support |

## 8. Form Validation

### 8.1 Validation Rules by Field Type

| Field Type | Validations | Error Messages |
|------------|-------------|----------------|
| Email | Required, email format | "Email is required", "Please enter a valid email" |
| Password | Required, min 8 chars, complexity | "Password must be at least 8 characters" |
| Select | Required, valid option | "Please select an option" |
| Date | Required, valid date, range | "Please enter a valid date" |
| Number | Required, min/max, integer | "Please enter a number between X and Y" |
| File | Required, size, type | "File must be PDF, JPG, or PNG (max 50MB)" |
| Text | Required, min/max length | "This field is required" |

### 8.2 Real-Time Validation

| Event | Validation Timing |
|-------|-------------------|
| On blur | Validate field when user leaves |
| On change | Validate after 500ms debounce |
| On submit | Validate all fields, scroll to first error |
